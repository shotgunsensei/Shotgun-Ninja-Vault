You are working in the existing Shotgun Ninja Vault (SNV) repo (Express+TS server, Vite+React client, Postgres+Drizzle). Do NOT change frameworks or refactor modules unnecessarily.

GOAL: Implement Stripe subscription monetization + plan enforcement for tenants.

High-level requirements:
- Use Stripe Billing subscriptions via Stripe Checkout for new subscriptions and Customer Portal for changes/cancellations.
- Maintain subscription status per tenant in DB, updated via Stripe webhooks.
- Enforce plan gates/limits in the server using a middleware that runs before module routes.

Stripe implementation notes:
- Stripe webhook signature verification requires the RAW request body. Create a dedicated Express route using express.raw({type:'application/json'}) and do not run JSON body parsing on that route.
- Use the official stripe Node SDK. Keep code minimal and documented.

B) tenantSubscriptions
- id (uuid pk)
- tenantId (fk tenants unique)
- stripeCustomerId (text nullable)

- stripePriceId (text nullable)
- planCode (text)  // references subscriptionPlans.code
- status (text enum: trialing|active|past_due|canceled|incomplete|incomplete_expired|unpaid)
- currentPeriodEnd (timestamp nullable)
- cancelAtPeriodEnd (boolean default false)
- createdAt, updatedAt

C) usageCountersMonthly
- id (uuid pk)
- tenantId (fk)
- monthKey (text) // YYYY-MM
- reportsGenerated (int default 0)
- webhookDeliveries (int default 0)

- createdAt, updatedAt
Unique (tenantId, monthKey)

3) Server routes (authenticated, OWNER/ADMIN):
- GET /api/billing/plans -> returns available plans + current tenant subscription
- POST /api/billing/checkout-session -> body {planCode} creates Stripe Checkout session for subscription
  - success_url = APP_URL + "/billing/success"
  - cancel_url = APP_URL + "/billing/cancel"
  - include tenantId in Stripe metadata
- POST /api/billing/customer-portal -> returns a Stripe billing portal URL
- GET /api/billing/subscription -> returns tenantSubscriptions row

4) Stripe webhook endpoint (public):
- POST /api/stripe/webhook using express.raw({type:'application/json'})
- Verify signature using STRIPE_WEBHOOK_SECRET.
- Handle events at minimum:
  - checkout.session.completed (create customer/subscription mapping)
  - customer.subscription.created/updated/deleted (update tenantSubscriptions)
  - invoice.payment_failed (set past_due)
  - invoice.paid (set active)
- Use Stripe event’s metadata. Ensure tenantId mapping is correct and tenant isolated.
- Emit SNV events via emitEvent:
  billing.subscription_updated
  billing.payment_failed

5) Plan enforcement middleware:
Create server/core/billing/enforcePlan.ts that:
- Loads tenant plan + limits
- Provides helper guard functions:
  requireFeature('api'|'portal'|'status'|'webhooks'|'reports')
  checkLimit('usersMax'|'webhooksMax'|'reportsPerMonth'|'storageGb')
- Apply enforcement in server/routes.ts BEFORE registering module routes:
  - API module: blocked if apiEnabled=false
  - Portal module: blocked if portalEnabled=false
  - Status module: blocked if statusEnabled=false
  - Webhooks: prevent creating more endpoints than webhooksMax
  - Reports: prevent generating more than reportsPerMonth in current monthKey
- If blocked, return 402 Payment Required with JSON {error:'plan_required', requiredPlan:'pro', feature:'api'}
